Metadata doorgeven ()
Doel - Het specificeren van eisen aan berichten om verschillende typen meta informatie door te geven. Belang voor elektronische gegevensuitwisseling - Om gegevens goed te duiden is metainformatie minstens zo belangrijk als het gegeven zelf. Als een getal op zichzelf staat b.v. 57, zonder een unit, b.v. kg, is het zinloos om door te geven. Een bloeddruk van 80/45 mm HG zonder de context van de patient (post OK volwassene of baby?) is het niet te interpreteren. Meta informatie zorgt voor contexten. Metadata gaan over concrete gegevens (zibs), gaan over de zorgsituatie, het bericht, de use case, volgordelijkheid (MBH nummer uit MP9). Metadata gaan soms ook om relaties tussen gegevens. Op dit moment zijn er geen eisen voor gespecificeerd. B.v. in een berichtvoorbeeld zijn 1000 XML regels opgenomen om een complete zib te specificeren op basis waarvan 3 regels gegevens worden doorgegeven. Door alleen de unieke OID van de zib op te nemen als meta informatie kunnen 997 regels XML verdwijnen en wordt uitwisseling eenvoudiger. Bestaande initiatieven -
